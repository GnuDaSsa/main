# 오케이

## 트리거
- 사용자가 `오케이` 스킬을 명시할 때
- 또는 사용자가 "수정 후 확인하고 커밋" 흐름을 요청할 때

## 핵심 규칙
- 코드 수정/실행/테스트는 진행한다.
- 변경 요약과 검증 결과를 먼저 보고한다.
- 사용자가 `오케이`라고 명시하기 전에는 `git commit`, `git push`를 실행하지 않는다.
- 사용자가 `오케이`라고 답하면 그때 **커밋 + 원격 반영(push)** 까지 수행한다.
  - 기본값: `origin/main`으로 push해서 배포 서버에 반영되게 한다.
  - 예외: 사용자가 "커밋만"이라고 명시하면 push는 하지 않는다.

## 실행 절차
1. 요구사항을 반영해 파일을 수정한다.
2. 가능한 테스트/실행 검증을 수행한다.
3. 다음 형식으로 보고한다.
- 변경 파일
- 핵심 변경점
- 검증 결과
- 커밋 대기 상태 (`승인 대기: 오케이`)
4. 사용자가 `오케이`라고 말하면:
   - 변경 범위를 재확인(의도하지 않은 파일/로그/캐시 제외)
   - 커밋 수행
   - `origin/main`에 push (사용자가 "커밋만"이라고 한 경우는 제외)
   - 원격 HEAD/커밋 해시를 확인해 보고

## 커밋 메시지 규칙
- 간결한 현재형 한 줄 사용
- 예: `Update landing page layout and text`

## 실무 가드레일
- 커밋/푸시 대상은 기본적으로 **의도한 소스 파일만** 포함한다(예: `app.py`, `main/app.py`).
- `streamlit.log`, `*.err`, `__pycache__/` 같은 런타임 산출물은 포함하지 않는다.
- 로컬 `git`이 고장난 환경이면, 다음 우선순위로 원격 반영을 수행한다:
  1) `git` CLI로 `commit` + `push`
  2) (대안) 파이썬 Git 라이브러리로 `commit` + `push`
  3) (최후) GitHub API로 **파일 내용을 원격 커밋으로 직접 반영**
