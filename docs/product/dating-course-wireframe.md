# Dating Course Wireframe (Text)

## 1. Screen A: Input
- 헤더: "우리 둘 사이, 무난한 데이트 코스"
- 필드:
  - 남자 출발역 (자동완성)
  - 여자 출발역 (자동완성)
  - 만남 시간
  - 예산대 (`low/normal/high`)
  - 이동수단 (`transit/car_optional`)
  - 분위기 (`neutral`, 기본 고정)
- CTA: `코스 추천받기`

## 2. Screen B: Result List
- 카드 3개 노출
- 카드 요소:
  - 코스명
  - 총 소요시간
  - 이동시간 격차
  - 예산대
  - 주차 상태 배지
  - 추천 근거 3줄
- CTA:
  - `상세 보기`
  - `비슷한 코스 다시 추천`

## 3. Screen C: Course Detail
- 상단 요약:
  - 코스 점수
  - 이동 공정성(격차 분)
  - 실시간 주의 배지(혼잡/영업시간)
- 타임라인:
  - 식사 -> 카페 -> 활동
- 각 스팟 상세:
  - 위치 구역
  - 후기 요약
  - 대기/혼잡 리스크
  - 주차 정보(차량 모드 시)
- 하단 CTA:
  - `코스 저장`
  - `공유`

## 4. UX Rules
1. 단순 거리 추천 문구 금지.
2. 모든 코스에 "왜 이 코스인지" 설명 필수.
3. 주차 정보 불확실 시 "확인 필요" 배지 노출.
4. 추천 실패 시 후보 구역 재탐색 후 재시도.
