# Dating Course Recommender PRD (v1.0)

## 1. Product Summary
- 서비스명(가칭): Between Us
- 목표: 남/여 출발 지하철역을 입력하면 수도권 내 데이트 만족도가 높은 코스 3개를 추천한다.
- 핵심 원칙: 정중앙 최적화가 아닌 만족도 최적화.

## 2. Target Users
- 20~30대 수도권 사용자
- 만남 장소 합의에 시간이 오래 걸리는 커플/소개팅 사용자

## 3. Problem Statement
- 단순 거리 기준 추천은 실제 데이트 만족도를 반영하지 못한다.
- 사용자들은 이동 공정성, 분위기, 대기 리스크, 주차 가능 여부까지 함께 고려한다.

## 4. Scope
### In Scope (MVP)
- 입력: 남자역, 여자역, 시간대, 예산대, 이동수단(대중교통/차량 가능), 분위기(기본 무난)
- 출력: 코스 3개 (식사-카페-활동), 코스별 근거/이동격차/예산/주차 상태
- 후기 기반 만족도 신호 반영
- 실시간성(영업 상태/혼잡 추정/이동시간) 반영

### Out of Scope (MVP)
- 결제/예약
- 실시간 주차 잔여면 100% 보장
- 전국 범위 확장

## 5. Ranking Policy (MVP)
- 점수식:
  - `Score = 0.35 review + 0.20 neutral_mood + 0.15 realtime + 0.15 fairness + 0.10 access + 0.05 parking`
- 차량 미사용 시 parking 가중치는 review/fairness/realtime로 재분배.
- 이동시간 격차가 과도한 코스는 하드 필터로 제외.

## 6. Functional Requirements
1. 역 입력 자동완성(수도권 지하철역).
2. 후보 구역 3~6개 생성.
3. 코스 조합 생성 및 스코어링.
4. 상위 3개 코스 반환.
5. 코스별 설명(왜 추천했는지) 제공.

## 7. Non-functional Requirements
- 응답시간 p95 5초 이하
- 추천 실패율 2% 미만
- 위치 개인정보 최소 수집(입력 역 중심)

## 8. KPI
- 코스 상세 진입률
- 코스 저장/공유율
- 사용자 만족 응답률
- 7일 재방문율

## 9. Milestones
1. M1: 요구사항/데이터 소스 확정
2. M2: 추천 엔진 MVP
3. M3: 후기 신호 정제 + 실시간 반영 강화
4. M4: 베타 검증 및 튜닝
